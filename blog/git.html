Version control helps track changes, and easily corrects costly errors, by going back to a previous, more stable version. It can help identify when, and who, made what changes as well as allow people to simultaneously collaborate on a single project.

Git is a very simple and fast way of tracking changes within a shared folder. A user can either group changes together, or publish them indivdually. Either way commiting changes to files is a great way to keep track of your changes, as well as recall them at a later date.

Github a great way to track changes in your code and already has a large user base and expansive helpful community. It's a great way to check, test, peer review, and submit your code without damaging or even altering the master code.
